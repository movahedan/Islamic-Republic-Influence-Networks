# Islamic Republic Influence Networks

AI-focused project context. For human overview and dataset description see [README.md](README.md). For code standards and workflows see [.cursor/rules/](.cursor/rules/).

## What this repo is

- **Dataset**: Public OSINT account metadata (X/Twitter, Instagram) for research on influence patterns (IR-related, MEK, White Internet). Stored in `Data/*.json`.
- **Extension**: Chrome MV3 content script for x.com/twitter.com that shows category badges and retweet summaries for handles present in the built dataset. Not a watchlist; research/journalism use only.

## Commands

Run from **repo root** unless noted.

| Command | Where | Purpose |
|--------|--------|---------|
| `bun install` | root | Install deps for workspace (root + extension). Runs `lefthook install` postinstall. |
| `bun run check` | **extension/** | Lint with Biome. |
| `bun run check:fix` | **extension/** | Biome check and fix. |
| `bun run check:types` | **extension/** | TypeScript check (`tsc --noEmit`). |
| `bun run build:handles` | **extension/** | Regenerate `extension/src/data/handles.json` from `Data/*.json`. |
| `bun run build` | **extension/** | `build:handles` then Vite build. Output: `extension/dist/`. |
| `bun run dev` | **extension/** | `build:handles` then Vite watch. |
| `bun run deploy` | **extension/** | Build and zip extension for store/release. |

**Workflow (from [workflow.mdc](.cursor/rules/workflow.mdc)):** Before commit run in **extension/**:

```bash
bun run check:fix
bun run check:types
bun run build
```

## Extension architecture (content script)

Entry: `extension/src/content/index.ts`. No background or popup.

- **Store** (`content/store.ts`): Single reducer-style store. State: theme, profileHeaderBadgeHost, retweetCountById, retweetSeenHandles, retweetSummaryHost, retweetSummaryUpdate, retweetPathname. Actions: SET_THEME, SET_PROFILE_HEADER_BADGE_HOST, SYNC_PROFILE_HEADER_BADGE_HOST, SET_RETWEET_*, CLEAR_RETWEET_SUMMARY. Theme synced from DOM via themeObserver.
- **Mutators** (`content/mutators.tsx`): Declarative list of `{ id, condition, getItems?, getOptions, mutate }`. Four mutators: profile-header-badge, user-cell-badge, tweet-article-badge, retweets-summary. `shouldScan(node)` excludes nodes with `data-testid^='ir-'`. User-cell mutator updates retweet counts; retweets-summary mutator mounts/updates the summary UI. Mutators use `render()` from lib and dispatch to store.
- **Observers** (`lib/observers.ts`): `themeObserver(callback)` — syncs theme from `data-theme` on documentElement; `routeChangeObserver(callback)` — popstate + interval, used to dispatch SYNC_PROFILE_HEADER_BADGE_HOST (store removes profile header badge on route change); `throttledObserve(mutators, shouldScan)` — MutationObserver, throttle 80ms, runs mutators on roots where `shouldScan`.
- **Selectors** (`lib/selectors.ts`): DOM query helpers (Theme, ProfileHeader, HandleLink, TweetArticle, UserName, UserCell, UserDashName, RetweetsPage*, etc.).
- **Utilities** (`lib/utilities.ts`): `isProfilePage()`, `isRetweetsPage()`, `getHandleFromUrl()`, `getHandleFromElement()`.
- **Categories** (`lib/categories.ts`): Reads `handles.json`. `getCategory(handle)` → `Category | null`, `getCategoryById(categoryId)` → `Category | null`. `categoryVariants` in file (e.g. ir-x-network → gray, mek → red).
- **Render** (`lib/render.ts`): `render(anchor, component, options)` — creates Shadow-DOM host, mounts React root, returns `{ host, mount, root, update }`. Used by mutators to inject Badge and RetweetsSummary.
- **Components** (`lib/components.tsx`): React — `Badge`, `RetweetsSummary`.
- **Types** (`lib/types.ts`): `Theme`, `Category`, `Anchor`, `Mutator`, `MutatorOptions`.

Data flow: index runs themeObserver → store SET_THEME; routeChangeObserver → store SYNC_PROFILE_HEADER_BADGE_HOST (store removes badge if still connected); throttledObserve(mutators, shouldScan) runs mutators on DOM changes. Mutators read store, call render() with components, and dispatch actions.

## Customization

- **Datasets**: In `extension/scripts/build-handles.ts`, edit `files` (e.g. add `White-Internet.json`) and `categoryLabels`. File name (minus `.json`) becomes category id (lowercase, underscores → hyphens).
- **Category labels/variants**: Build script sets labels; `extension/src/lib/categories.ts` sets `categoryVariants` (gray vs red). Red affects styling and any future “red” notice behavior.
- **Selectors**: X’s DOM changes often. Update `extension/src/lib/selectors.ts` and helpers in `extension/src/lib/utilities.ts` when the site markup changes.
