# Influence Network Tags (Local)

A **Chrome MV3** content-script extension for **x.com** and **twitter.com**. It shows a neutral research label next to accounts that appear in this repo’s public OSINT datasets (e.g. MEK, IR-X-Network). Labels are category-specific (e.g. “OSINT Flagged”, “Mojahedin”) and appear on timelines, profile headers, and retweet summaries.

**Not a watchlist or targeting tool.** For research and journalism only; see the upstream repo README for responsible-use terms.

---

## What it does

- **Timeline & profile:** Injects a small badge next to the @handle when the account is in the built dataset.
- **Tweets:** For “red” categories, adds a short notice under the tweet (e.g. “Check the profile details before getting engaged”).
- **Retweets page:** Shows a summary of how many retweets are from listed accounts per category.
- **Theme:** Badges and notices follow X’s light/dark theme.

Handles are matched from the first profile-like `/username` link in each tweet or user cell. Data comes from `../Data/*.json`; the extension bundle includes a pre-built `handles.json` generated by the build script.

---

## Build and load

From the **extension** directory:

```bash
bun install
bun run build
```

In Chrome: open **Extensions** → turn on **Developer mode** → **Load unpacked** → choose `extension/dist`.

---

## Scripts

| Script | Description |
|--------|-------------|
| `bun run build:handles` | Regenerates `src/data/handles.json` from `../Data/*.json` (skips missing files). Handles are normalized (lowercase, `@` stripped). |
| `bun run build` | Runs `build:handles` then builds the Vite bundle. |
| `bun run dev` | Runs `build:handles` then Vite in watch mode. |
| `bun run deploy` | Builds and zips the extension into `dist/<name>-v<version>.zip` for Chrome Web Store or releases. |
| `bun run check` | Lint with Biome. |
| `bun run check:types` | Type-check with `tsc --noEmit`. |

---

## Customization

- **Datasets:** In `scripts/build-handles.ts`, edit the `files` array to add or remove JSON sources (e.g. `IR-X-Network.json`, `MEK.json`). Each file’s name (without `.json`) becomes a category id (lowercase, underscores → hyphens).
- **Category labels:** In the same script, edit `categoryLabels` to change the text shown in the UI (e.g. `"ir-x-network": "OSINT Flagged"`).
- **Which categories are “red” vs “gray”:** In `src/lib/categories.ts`, adjust `categoryVariants` (e.g. `"mek": "red"`). Red categories get the extra tweet notice and distinct styling.

---

## Notes

- X’s DOM changes frequently. Selectors (e.g. `data-testid="User-Name"`) in `src/lib/selectors.ts` may need updates when the site changes.
- Matching is best-effort; false positives and false negatives are possible.
- The extension is content-script only (no background or popup). It runs on `https://x.com/*` and `https://twitter.com/*`.
